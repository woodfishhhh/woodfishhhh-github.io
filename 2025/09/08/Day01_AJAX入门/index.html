Day01_Ajax 入门

一、AJAX 核心概念

1.1 什么是 AJAX

AJAX(Asynchronous JavaScript And XML)是一种使用现有标准的 Web 开发方法，可以在不重新加载整个页面的情况下，与服务器交换数据并更新部分网页内容。

主要特点：

  • 异步通信：在不重新加载整个页面的情况下与服务器通信
  • 局部刷新：可以实现页面的局部更新
  • 基于 XML：最初设计时使用 XML 格式传递数据（现在主要使用 JSON）

1.2 为什么使用 AJAX

  1. 提升用户体验

    • 无需刷新整个页面
    • 快速响应用户操作

  2. 减轻服务器负担

    • 按需获取数据
    • 局部更新减少数据传输

  3. 实现实时交互

    • 在线聊天
    • 数据实时更新

1.3 技术核心与组成

  1. 核心对象

    • XMLHttpRequest 对象(XHR)
    • Axios(基于 XHR 封装的库)

  2. 数据格式

    • JSON(主流)
    • XML(传统)
    • FormData

  3. 服务器通信

    • HTTP 协议
    • RESTful API

四、HTTP 通信基础

4.1 URL 结构解析

基本概念

URL(Uniform Resource Locator)统一资源定位符，是互联网上资源的唯一标识。

URL 组成部分

  1. 协议(protocol)：如 http、https
  2. 主机名(host)：如www.example.com
  3. 端口号(port)：如 80、443
  4. 路径(path)：如/api/users
  5. 查询参数(query string)：如?id=1&name=test
  6. 片段标识符(fragment)：如#section1

URL 实例解析

  1. 协议：https://
  2. 主机名：api.example.com
  3. 端口号：443
  4. 路径：/users/profile
  5. 查询参数：?id=123
  6. 片段标识：#photo

最佳实践

  1. 使用 HTTPS 协议保证安全性

  2. 采用 RESTful 风格设计 API

  3. 合理使用查询参数传递数据

  4. URL 需要进行编码处理特殊字符

  5. 什么是 URL ？

    • 统一资源定位符，简称网址，用于定位网络中的资源（资源指的是：网页，图片，数据，视频，音频等等）

  6. URL 的组成?

    • 协议，域名，资源路径（URL 组成有很多部分，我们先掌握这 3 个重要的部分即可）

  7. 什么是 http 协议 ?

    • 叫超文本传输协议，规定了浏览器和服务器传递数据的格式（而格式具体有哪些稍后我们就会学到）

  8. 什么是域名 ?

    • 标记服务器在互联网当中的方位，网络中有很多服务器，你想访问哪一台，就需要知道它的域名才可以

  9. 什么是资源路径 ?

    • 一个服务器内有多个资源，用于标识你要访问的资源具体的位置

4.2 HTTP 请求方法

常用请求方法

  1. GET：获取资源
  2. POST：创建资源
  3. PUT：更新资源
  4. DELETE：删除资源
  5. PATCH：部分更新

请求方法特点

  1. GET
    • 参数通过 URL 传递
    • 有长度限制
    • 可被缓存
  2. POST
    • 参数在请求体中
    • 无长度限制
    • 更安全
    • 不可被缓存

09.HTTP 协议-响应报文

目标

了解响应报文的组成

讲解

  1. 响应报文：是服务器按照协议固定的格式，返回给浏览器的内容

  2. 响应报文的组成：

    • 响应行（状态行）：协议，HTTP 响应状态码，状态信息
    • 响应头：以键值对的格式携带的附加信息，比如：Content-Type（告诉浏览器，本次返回的内容类型）
    • 空行：分割响应头，控制之后的是服务器返回的资源
    • 响应体：返回的资源

  3. HTTP 响应状态码：

    • 用来表明请求是否成功完成

    • 例如：404（客户端要找的资源，在服务器上不存在）

小结

  1. 响应报文的组成？

    答案
    • 响应行，响应头，空行，响应体

  2. HTTP 响应状态码是做什么的？

    答案
    • 表明请求是否成功完成，2xx都是成功的
