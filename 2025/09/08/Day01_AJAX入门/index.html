一、AJAX是什么？（用大白话解释）

1.1 生活中的AJAX例子

想象你去咖啡店：

  • 传统方式：每次点单都要重新排队、重新点单、重新付款（就像网页刷新）
  • AJAX方式：你只需要告诉服务员要加什么，服务员直接去准备，你不用重新排队（页面不刷新）

1.2 技术定义

AJAX = 异步的JavaScript和XML技术

  • 异步：不用等待，可以同时做其他事情
  • JavaScript：用JS语言实现
  • XML：最初用XML传数据（现在主要用JSON）

1.3 为什么要学AJAX？

传统网页 AJAX网页
每次操作都要刷新整个页面 只更新需要改变的部分
用户体验差，等待时间长 用户体验好，响应快速
服务器压力大 服务器压力小

实际应用场景：

  • 🔄 用户注册时实时检查用户名是否已存在
  • 💬 聊天应用实时接收新消息
  • 🛒 购物车添加商品无需刷新页面
  • 📧 邮箱实时接收新邮件

4.2 HTTP 请求方法

常用请求方法

  1. GET：获取资源
  2. POST：创建资源
  3. PUT：更新资源
  4. DELETE：删除资源
  5. PATCH：部分更新

请求方法特点

  1. GET
    • 参数通过 URL 传递
    • 有长度限制
    • 可被缓存
  2. POST
    • 参数在请求体中
    • 无长度限制
    • 更安全
    • 不可被缓存

09.HTTP 协议-响应报文

目标

了解响应报文的组成

讲解

  1. 响应报文：是服务器按照协议固定的格式，返回给浏览器的内容

  2. 响应报文的组成：

    • 响应行（状态行）：协议，HTTP 响应状态码，状态信息
    • 响应头：以键值对的格式携带的附加信息，比如：Content-Type（告诉浏览器，本次返回的内容类型）
    • 空行：分割响应头，控制之后的是服务器返回的资源
    • 响应体：返回的资源

  3. HTTP 响应状态码：

    • 用来表明请求是否成功完成

    • 例如：404（客户端要找的资源，在服务器上不存在）

小结

  1. 响应报文的组成？

    答案
    • 响应行，响应头，空行，响应体

  2. HTTP 响应状态码是做什么的？

    答案
    • 表明请求是否成功完成，2xx都是成功的
